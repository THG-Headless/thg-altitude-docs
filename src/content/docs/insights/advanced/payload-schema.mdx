# Payload Schema

The Insights SDK collects and transmits analytics data in three categories: page views, events, and sessions. This page documents the structure of this data for developers who need to understand what is being collected.

## Data Categories

### Page Views

Captured automatically when a user visits a page. Includes device information, attribution data, and performance metrics.

**When sent:** Immediately on page load

### Events

User interactions tracked via HTML attributes or the JavaScript API. Includes clicks, impressions, and custom events.

**When sent:** Batched every 30 seconds, or when the batch reaches 50KB, or on page exit

### Sessions

Engagement data tracking how users interact with the page over time. Sessions capture rich interaction data including viewport changes, scroll behaviour, and tab visibility.

**When sent:** On visibility change (tab hidden) and page exit

**Session data includes:**

| Field | Description |
|-------|-------------|
| `session_index` | Increments each time the tab becomes visible again |
| `start_time` | When this session segment started |
| `end_time` | When this session segment ended |
| `viewports` | Array of viewport dimensions with duration at each size |
| `document_states` | Array containing document height snapshots over time |
| `max_scroll_depth` | Deepest scroll position reached (pixels) |
| `scroll_events` | Number of scroll interactions |

Performance metrics (Core Web Vitals) are included with the final session on page exit.

## Event Structure

Events tracked via `data-insights-*` attributes or the JavaScript API follow this structure:

```typescript
interface Event {
  event_category: string; // 'click', 'impression', or custom type
  event_data: {
    [key: string]: unknown; // Flexible key-value pairs
  };
  timestamp: number; // Unix timestamp in milliseconds
}
```

### Example Events

**Click event from HTML attributes:**

```json
{
  "event_category": "click",
  "event_data": {
    "category": "button",
    "id": "signup-cta",
    "action": "signup"
  },
  "timestamp": 1704067260000
}
```

**Custom event from JavaScript API:**

```json
{
  "event_category": "purchase",
  "event_data": {
    "category": "ecommerce",
    "id": "order-123",
    "value": 99.99,
    "currency": "GBP"
  },
  "timestamp": 1704067265000
}
```

## Event Batching

Events are collected in memory and sent in batches for efficiency. A batch is sent when any of these conditions are met:

- **Time threshold:** 30 seconds since the last batch
- **Size threshold:** Batch exceeds 50KB
- **Page exit:** User navigates away or closes the tab

Events are sent using the Beacon API to ensure reliable delivery even during page unload.

## Page View Data

Page views include contextual information about the visit:

| Field | Description |
|-------|-------------|
| `path` | URL path of the page |
| `url` | Full page URL |
| `title` | Page title |
| `referrer` | Referring URL |
| `domain` | Site domain |
| `viewport` | Browser viewport dimensions |
| `timestamp` | When the page was loaded |
| `channel` | Channel identifier from configuration |
| `subsite` | Subsite identifier from configuration |
| `storefront` | Storefront identifier from configuration |
| `pageType` | Page type if configured via attribute or init |
| `navigationType` | How the page was reached (`navigate`, `reload`, `back_forward`, `prerender`) |

### Query Parameters

All query parameters from both the page URL and referrer URL are captured automatically. This includes common attribution parameters such as:

- UTM parameters (`utm_source`, `utm_medium`, `utm_campaign`, etc.)
- Ad platform click IDs (`gclid`, `fbclid`, etc.)
- Any other query parameters present in the URL

### Performance Metrics

Core Web Vitals are collected when available:

| Metric | Description |
|--------|-------------|
| FCP | First Contentful Paint |
| LCP | Largest Contentful Paint |
| CLS | Cumulative Layout Shift |
| INP | Interaction to Next Paint |
| TTFB | Time to First Byte |

## Data Linking

All data is linked together using identifiers:

- **Page UUID:** Links events and sessions to their originating page view
- **Session Cookie:** Links page views across a browsing session
- **User Cookie:** Links sessions across multiple visits

You can access the current page UUID programmatically:

```javascript
const pageUUID = window.insights.getPageUUID();
```
