# Installation

Get started with Insights by installing the NPM package and importing the appropriate variant for your application type.

## NPM Package (Recommended)

Install via NPM for modern build systems:

```bash
npm install @thg-altitude/insights
```

## Choosing the Right Build

Then import the appropriate variant in your JavaScript/TypeScript files:

The insights package provides 4 variants, as seen below. The split between MPA and SPA behaviour results in optimised builds for each use case, while the developer and production variants provide the choice between the silent production build and the verbose debug build, allowing for a minimal bundle size in production and detailed logging during development.

| Variant            | Import                             | Bundle Size |
| ------------------ | ---------------------------------- | ----------- |
| **MPA Production** | `@thg-altitude/insights/mpa`       | ~30KB       |
| **MPA Debug**      | `@thg-altitude/insights/mpa/debug` | ~120KB      |
| **SPA Production** | `@thg-altitude/insights/spa`       | ~31KB       |
| **SPA Debug**      | `@thg-altitude/insights/spa/debug` | ~122KB      |

## Adding the Component

The appropriate component should be imported within the main template used across your application, such as the `Layout.astro` file. It is important to ensure that this component is loaded as quickly as possible, and as such should be placed within the `<head>` tag.

```javascript
<!DOCTYPE html>
<html>
  <head>
    <insights-tracker tenant={tenantConfig?.siteName || 'altitude'}></insights-tracker>
  </head>
  <body>
    <!-- Your content -->
  </body>
</html>
```

## Verification

After importing, we can verify the installation by checking for the presence of the global `insights` object via the browser console:

```javascript
// Check if insights is available
if (typeof window.insights !== "undefined") {
  console.log("✅ Insights loaded successfully");
} else {
  console.error("❌ Insights failed to load");
}
```

We should also see network events fire on navigation.

> **Note**: In some browsers such as chrome, since the events are fired using the beacon api, following a fire and forget approach, the events may appear to fail in the network tab. This is expected behaviour, and a quirk of the browser.
