# Component Configuration

Once the component is added to your codebase, it can be configured through its props, as documented below.

## Configuration Options

| Attribute           | Required | Description                                                          |
| ------------------- | -------- | -------------------------------------------------------------------- |
| `tenant`            | ✅       | Your company identifier for multi-tenant data segregation            |
| `base-url`          | ❌       | Default base URL for all endpoints (e.g., `https://api.example.com`) |
| `auth-endpoint`     | ❌       | Override URL for authentication endpoint                             |
| `pageview-endpoint` | ❌       | Override URL for page view tracking endpoint                         |
| `event-endpoint`    | ❌       | Override URL for event tracking endpoint                             |

## Required Attributes

Only the tenant attribute is required, as this is necessary for differentiating between clients in multi-tenant environments, enabling data segregation within the tracking pipeline.

```html
<insights-tracker tenant="your-company-name"></insights-tracker>
```

## Optional Attributes

In addition to the required tenant attribute, the following optional attributes can be used to customize the behavior of the component:

```html
<insights-tracker
  tenant="your-company-name"
  base-url="https://custom.example.com"
  auth-endpoint="https://custom.example.com/auth"
  pageview-endpoint="https://custom.example.com/pageviews"
  event-endpoint="https://custom.example.com/events"
>
</insights-tracker>
```

#### Endpoint Configuration Behavior

The endpoint attributes work together in the following way:

**`base-url`** - Sets the default base URL for all endpoints

- When specified alone, all endpoints will use this base URL with their default paths
- Example: `base-url="https://api.example.com"` results in:
  - Auth: `https://api.example.com/auth`
  - Page views: `https://api.example.com/pageviews`
  - Events: `https://api.example.com/events`

**Individual endpoint URLs** - Override the base URL for specific endpoints

- When specified, they completely replace the URL for that endpoint
- They are absolute URLs and ignore the `base-url` setting
- Example with mixed configuration:
  ```html
  <insights-tracker
    tenant="company"
    base-url="https://api.example.com"
    event-endpoint="https://events.different.com/track"
  ></insights-tracker>
  ```
  Results in:
  - Auth: `https://api.example.com/auth` (uses base-url)
  - Page views: `https://api.example.com/pageviews` (uses base-url)
  - Events: `https://events.different.com/track` (overrides with absolute URL)

**Default behavior** - When no URLs are specified

- Uses the default Insights service endpoints
