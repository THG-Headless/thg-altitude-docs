# Event Tracking

While page views will automatically be tracked once the insights component has been added to your codebase, the Insights package additionally provides an event stream from which front end events can be tracked. HTML `data-insights-*` attributes can be used for tracking events as seen below, providing a quick and easy way to add tracking to any component in the codebase without the need for any additional logic. The JavaScript API is also exposed, although this should be used as a last resort when the HTML attributes are insufficient and the event does not have a corresponding back end event.

## HTML Attribute Tracking

In order to trigger a front end event, the following attributes are required on the relevant HTML element:

- `data-insights-track` - The type of tracking to perform. Currently supports `click` and `impression`.
- `data-insights-category` - The category of the element being tracked (e.g., button, navigation).
- `data-insights-id` - A unique identifier for the element being tracked.

In addition to these, any additional data can be provided using custom `data-insights-*` attributes.

### Click Tracking

Applying `data-insights-click` attributes to your HTML elements allows for automatic tracking of clicks.

Here we see an example of a signup button, where the click event is tracked:

```html
<button
  data-insights-track="click"
  data-insights-category="button"
  data-insights-id="signup-cta"
>
  Sign Up
</button>
```

This would result in the following event object being submitted to the events stream when clicked:

```json
{
  "event_category": "click",
  "event_data": {
    "category": "button",
    "id": "signup-cta"
  },
  "timestamp": 1704067260000
}
```

#### Custom attributes

Clicks, along with all other types of events support the addition of custom attributes. These follow the pattern `data-insights-*`. As an example, we could use:

```html
<button
  data-insights-track="click"
  data-insights-category="button"
  data-insights-id="signup-cta"
  data-insights-action="signup"
>
  Sign Up
</button>
```

Which would result in the `action` property being added to the `event_data` object in the event payload.

```json
{
  "event_category": "click",
  "event_data": {
    "category": "button",
    "id": "signup-cta",
    "action": "signup"
  },
  "timestamp": 1704067260000
}
```

### Impression Tracking

Like clicks, we can track impressions in the same way. These events are triggered when the specified element becomes visible in the viewport.

```html
<div
  data-insights-track="impression"
  data-insights-category="product"
  data-insights-id="featured-product"
>
  <h3>Protein Shake</h3>
  <p>High-performance protein shake for athletes ...</p>
</div>
```

This would result in an event object like seen below being submitted to the events stream when the element is visible:

```json
{
  "event_category": "impression",
  "event_data": {
    "category": "product",
    "id": "featured-product"
  },
  "timestamp": 1704067260000
}
```

### Custom Event Tracking

Track custom events beyond clicks and impressions. Events follow the same structured format as HTML-tracked events for consistency.

```javascript
// Track a hover event
window.insights.event("hover", {
  category: "product",
  id: "featured-product",
  price: "19.99",
  position: "above-fold",
});

// Track form submission
window.insights.event("form_submit", {
  category: "form",
  id: "newsletter-signup",
  success: true,
  fields_completed: 3,
});

// Track video interaction
window.insights.event("video_play", {
  category: "video",
  id: "product-demo",
  duration: 120,
  position: 30,
});
```

**Parameters:**

- `type` (string): Event category (e.g., 'hover', 'form_submit', 'video_play')
- `eventData` (object): Event data with `category`, `id`, and custom properties

> Note that the type and event data are flexible. Type accespts any string, and event data can include any custom properties as needed.

**Returns:** `Promise<boolean>` - Success status

This produces events like:

```json
{
  "event_category": "hover",
  "event_data": {
    "category": "product",
    "id": "featured-product",
    "price": "19.99",
    "position": "above-fold"
  },
  "timestamp": 1704067260000
}
```

### Getting Page UUID

Get the current page UUID that links events to page views.

```javascript
const pageUUID = window.insights.getPageUUID();
console.log("Current page UUID:", pageUUID);
```

**Returns:** `string | null` - Current page UUID
